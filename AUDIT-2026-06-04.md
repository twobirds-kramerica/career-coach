## Anti-Patterns Verdict

Mostly passes the "AI made that" test — DM Sans + DM Serif Display is distinctive, dark-navy palette is intentional, layout is purposeful. Two explicit bans are present and a third pattern is encroaching:

1. **Side-stripe ban violated x2.** `.market-insights-panel { border-left: 4px solid var(--teal) }` (line 405) and `.industry-insights-card { border-left: 4px solid var(--amber) }` (line 839).
2. **Hero-metric template encroaching.** Job search stats grid (`.stat-value` at 1.6em bold teal, small uppercase label) is the big-number/small-label SaaS dashboard cliché. Not over the line yet.
3. **Modal overuse.** 6 modals for content that could live inline in the job detail panel.

## Detailed Findings

### P1 — Must Fix Before Release

**[P1] `--bg` token undefined — stat items transparent**
- Location: `:root` (line 23–38); `.stat-item { background: var(--bg) }` (line 186)
- Category: Theming
- Impact: All job search stat items (total, avg fit, recommended, skipped) render with no background. Visual regression.
- Fix: Add `--bg: #0F1B2D;` to `:root`

**[P1] DM Sans not in `body` font stack**
- Location: `body { font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto... }` (line 40)
- Category: Theming
- Impact: DM Sans is self-hosted and preloaded but body copy renders in San Francisco/Segoe UI. PRODUCT.md says "Font: DM Sans... correct and distinctive. Maintain." This is a brand breach.
- Fix: `body { font-family: 'DM Sans', -apple-system, BlinkMacSystemFont, sans-serif; }`

**[P1] No focus trap in modals**
- Location: All 6 modal overlays (lines 1408–1478); settings panel (line 288)
- Category: Accessibility
- WCAG: 2.1.2 No Keyboard Trap (Level A)
- Impact: Keyboard users can tab into content behind the overlay. Focus is not moved to the modal on open, not locked while open, not returned to trigger on close.
- Fix: On open, move focus to close button; add keydown listener trapping Tab/Shift+Tab inside; on close, return focus to trigger element.

### P2 — Fix Before Release

**[P2] Form inputs not programmatically labelled in onboarding wizard**
- Location: Step 1 and Step 2 field groups (lines 978–1096)
- Category: Accessibility
- WCAG: 1.3.1, 4.1.2
- Impact: ~12 fields have `<label>` not connected to their input via `for` attribute. Screen readers won't announce labels on focus.
- Fix: Add `for="pName"` etc. to every label element in the wizard.

**[P2] No `focus-visible` styles on interactive elements**
- Location: `.job-card`, `.filter-btn`, `.btn-icon`, `.btn-primary`, `.btn-secondary`, `.history-item`, `.btn-g0`
- Category: Accessibility
- WCAG: 2.4.7 Focus Visible (Level AA)
- Impact: Keyboard users get no visible focus indicator on the vast majority of interactive elements.
- Fix: Add global `:focus-visible { outline: 2px solid var(--teal); outline-offset: 3px; }`

**[P2] Two banned side-stripe borders**
- Location: `.market-insights-panel { border-left: 4px solid var(--teal) }` (line 405); `.industry-insights-card { border-left: 4px solid var(--amber) }` (line 839)
- Category: Anti-Pattern
- Fix: Replace with full border at reduced opacity or background tint. Example: `background: rgba(0,201,167,0.04); border: 1px solid rgba(0,201,167,0.3);`

**[P2] Mobile touch targets below 44px minimum**
- Location: `.filter-btn` gets `min-height: 36px` at ≤480px (line 551); `.salary-modal-close { padding: 4px 10px }` (line 784); `.btn-icon { padding: 6px 10px }` (line 67)
- Category: Responsive
- Impact: Modal close buttons and header icon buttons are below usable thumb size on mobile.
- Fix: `min-height: 44px; min-width: 44px` on `.salary-modal-close`; bump filter buttons to 44px at mobile; increase `.btn-icon` padding.

**[P2] `width` layout animation on hero headline underline**
- Location: `.hero-headline::after` `@keyframes underline-grow { to { width: 100% } }` (lines 336–344)
- Category: Performance
- Impact: Animating `width` triggers layout reflow on every frame.
- Fix: Use `transform: scaleX()` with `transform-origin: left center`. Set `width: 100%` statically, animate `scaleX(0)` to `scaleX(1)`.

**[P2] `--text-primary` / `--text-secondary` tokens declared but unused**
- Location: `:root` (lines 36–37)
- Category: Theming
- Impact: Duplicate tokens for `--warm` and `--muted` add confusion and rot.
- Fix: Remove `--text-primary` and `--text-secondary` from `:root`.

**[P2] Missing label: sort bar select and job input textarea**
- Location: `<label>Sort:</label>` (line 1201); `<textarea id="jobInput">` (line 1280)
- Category: Accessibility
- WCAG: 1.3.1, 4.1.2
- Fix: `<label for="sortSelect">Sort:</label>` and add `<label for="jobInput" class="sr-only">Job posting text</label>`

### P3 — Polish

**[P3] DM Sans Medium and SemiBold weights not preloaded**
- Location: `<head>` preload block (lines 16–17) — only Regular and Bold preloaded
- Category: Performance
- Fix: Add preload links for DMSans-Medium.woff2 and DMSans-SemiBold.woff2

**[P3] Hero metric pattern in stats grid approaching slop line**
- Location: `.job-stats-card` (lines 182–190)
- Category: Anti-Pattern
- Fix: Differentiate — one prominent hero number (total jobs), pipeline bar as visual track, other stats as supporting context in a different format.

**[P3] Wizard progress steps lack accessible labels**
- Location: `.wizard-step` divs (lines 965–969)
- Category: Accessibility
- Fix: Add `aria-label="Step 1: complete"` etc., updated dynamically in `goStep()`.

## Patterns and Systemic Issues

1. **Accessibility gaps are systemic in the onboarding wizard.** All ~12 form fields have the same label-association problem. One pass fixes all.
2. **Token discipline is inconsistent.** `#0F1B2D` appears hard-coded in 8+ places as an input/surface colour but has no token. Dead tokens co-exist with working ones.
3. **Modal pattern is the dominant interaction model.** 6 modals accumulated sprint-by-sprint. The job detail panel could host these as inline sections or accordions.

---

## Positive Findings

- Skip link implemented correctly — visible on focus, correct position and z-index, teal branded
- ARIA on primary modals is correct — `role="dialog"`, `aria-modal="true"`, `aria-labelledby` on all 5 AI tool modals
- Lazy API loading pattern is solid — Remotive and LFS data only fetch when panel is opened
- Responsive breakpoints are thorough — 768/700/600/480/380px; 16px textarea font-size prevents iOS zoom
- Gate 0 UX is distinctive — aggregator detection with `role="alert"` is genuinely useful and not done by competitors
- DM Serif Display used with restraint — headers only, not overused
- No glassmorphism, no gradient text, no bounce easing — the three most visible AI slop tells are absent
